# Skylite

We're a flight simulation network based in China, working to make the FSD ecosystem more open and accessible, so any network can have the same tooling and features that VATSIM has, without the proprietary lock-in.

## Projects

### Open source

- [**neoswift**](https://github.com/skylitefly/neoswift): Fork of swift pilot client with full per-network configurability, fsd-autodiscovery, and VATSIM proprietary components removed
- [**fsd-autodiscovery**](https://github.com/skylitefly/fsd-autodiscovery): Open standard for automated FSD network configuration.
- [**pyfsd**](https://github.com/skylitefly/pyfsd): Python implementation of the FSD server (for providing basic multiplayer connections)
- [**afv-server**](https://github.com/skylitefly/afv-server): Server implementation of AFV(audio for VATSIM) audio protocol
- [**slurper-api**](https://github.com/skylitefly/slurper-api): Part of AFV Protocol, provides user's realtime on-network data to controller client, supports standard whazzup.json(or vatsim-data.json) format
- [**weather-api**](https://github.com/skylitefly/weather-api): METAR/TAF HTTP API based on NOAA and avwx.rest data (our own deployment at https://weather.api.skylitefly.com )

### Coming soon

- **eaip-parser**: Parses China eAIP offline packages into a structured format for use by other services
- Our [portal](https://portal.skylitefly.com)'s frontend and backend
